# 项目前端交互规范（可执行）

本文件约束本项目的 UI/UX 交互与页面结构，目标是：流程清晰、反馈明确、交互一致、实现成本可控。

## 文案与提示

- 所有用户可见文案必须为中文（按钮/提示/占位/报错/空态）。
- 错误提示优先解释“下一步怎么做”，避免只报状态码或抽象错误。
- 上传/保存/删除等高风险操作必须有明确的状态反馈（进行中/成功/失败）。

## 推荐页面结构：两页流程

适用于“先选资源，再做配置/保存”的后台/工作台模块（例如任务模板、模板配置等）。

- 第 1 页：上传/选择页
  - 只做两件事：上传 PSD 或从列表选择已有模板
  - 上传成功 / 点击列表项后，自动进入第 2 页
- 第 2 页：配置/保存页
  - 顶部提供明确的“返回列表”
  - 聚焦配置与保存，避免列表噪音干扰
  - 返回列表不强制清空编辑状态（除非用户主动“新建/重置”）

## PSD 上传交互

- 必须同时支持：点击选择文件 + 拖拽放入。
- 拖拽态必须具备：
  - 区域高亮（描边/底色变化）
  - 覆盖提示（例如“松开即可上传 PSD”）
  - 拖拽离开后正确恢复（避免高亮卡死）
- 上传中必须阻止重复触发（按钮 disabled、拖拽阻止），并展示“正在上传并解析...”。
- 校验：
  - 选择非 PSD：提示“请上传 PSD 文件”
  - 拖入非 PSD：提示“请拖入 PSD 文件”

## 列表交互（模板选择）

- 搜索只做“包含匹配”（名称/ID），即时过滤，不增加确认按钮。
- 列表项 hover 反馈必须明显（背景/描边/阴影其一即可）。
- 当前激活项要更强对比（同时体现在背景与边框，避免只靠小标签）。
- 空态必须包含：
  - 原因（无数据/未匹配）
  - 下一步提示（例如：上传 PSD 新建、刷新列表）

## 视觉与动效（低成本）

- 暗色基底 + 单一强调色（例如 emerald/indigo），避免多色抢占注意力。
- 动效只服务“状态过渡”：hover、disabled、loading、进入/退出的轻微位移与透明度变化即可。
- 可点击区域足够大；禁用态必须明显；loading 必须有反馈。
