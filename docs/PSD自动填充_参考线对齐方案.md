# PSD 自动填充：参考线左右边缘对齐方案（草案）

## 1. 背景与目标

PSD 自动填充在做“图片变量替换”时，需要把输入商品图按预期对齐到商品位区域内。当前链路已具备“内容框提取 + 参考图对齐”的能力，但在一些素材（白底/近白噪点/主体形态变化大）场景中，算法提取的左右边缘不稳定，导致最终替换对齐不符合预期。

本方案目标是引入一套“设计侧显式标注左右边界”的机制，将图片对齐从“算法猜测”升级为“可控约束”，提升稳定性；并且做到：

- 不影响现有“文字无损替换导出”的功能与流程
- 对未添加参考线的模板完全保持现有行为（零回归）
- 仅在 PSD 自动填充分支的图片预处理阶段生效

## 2. 现状链路（简述）

服务端在 PSD 自动填充模式下，对图片变量会走内容边缘对齐逻辑：

- `server/services/photoshopIngest.js`：图片 update 写入前的预处理与落地
  - `persistAlignedImageToContentEdge(...)`
  - 调用 `imageProcessor.alignToRefContent(...)`
- `server/services/sharpProcessor.js`：Sharp 侧对齐实现
  - `alignToRefContent(...)` 当前主要依据“参考图在该商品位矩形内提取到的 refBounds”来计算目标区域（targetBox）

当前 `alignToRefContent` 不使用 guides/guideLayers，因此“只在 PSD 里加参考线”不会生效，需要在该函数内增加“参考线覆盖 targetBox”的分支。

## 3. PSD 制作规范（参考线如何画）

建议为每个需要替换的商品图变量（即 `{img:...}` 商品位）增加两条竖向参考线，分别标记商品内容的：

- 左边缘（Left）
- 右边缘（Right）

### 3.1 方案 A：使用 Photoshop 原生参考线（Guide）

在 PSD 中通过“视图 → 新建参考线”，在商品位区域内放置两条竖向 guide。

服务端对齐时，在“当前商品位矩形的 x 范围内”挑选竖向 guide，取最左与最右作为目标边界。

优点：不参与渲染、不影响导出、制作成本低。  
注意：guide 不具备命名与绑定能力，需避免在同一商品位 x 范围内存在多组无关 guide 造成歧义。

### 3.2 方案 B：使用参考线图层（Guide Layers）并支持绑定（推荐）

在 PSD 中增加一个“参考线组”，组名建议为：

- `参考线` 或 `__guides__`

在组内为每条参考线创建 1–2px 的竖向形状层/像素层，放在目标 x 位置，并将该组隐藏。

当前服务端识别参考线图层的默认条件是：

- 位于“参考线组”或图层名命中关键字（guide/参考线/辅助线/对齐线等）
- 宽度较小（约小于画布宽的 5%）
- 高度较大（约大于画布高的 30%）

为了保证可识别，建议参考线竖线做到足够高并隐藏。

### 3.3 绑定规则（消歧义，强烈建议）

为确保“每个商品位只命中自己的左右边界”，建议将参考线图层名带上图片变量图层的 `psId`：

- 左线：`psId_123_左`
- 右线：`psId_123_右`

服务端对齐某个 `psId=123` 的图片时，优先匹配带有 `psId_123` 的两条竖线；未找到时才回退到“商品位矩形范围内的最左/最右竖线”策略。

## 4. 服务端改动点（仅影响图片预处理）

仅在 `server/services/sharpProcessor.js` 的 `alignToRefContent(...)` 中增加一段“参考线覆盖”逻辑：

1) 先按现有逻辑计算 `targetBox`（保持现有行为）  
2) 若存在“可用的两条竖向参考线”：
   - 优先：按 `psId` 绑定匹配 `psId_{psId}_左/右`
   - 回退：在当前商品位矩形 x 范围内取最左与最右
3) 用参考线覆盖 `targetBox.left/targetBox.width`：
   - `targetBox.left = guideLeftX - rectAbsLeft`
   - `targetBox.width = guideRightX - guideLeftX`
4) `targetBox.top/height` 保持现有策略（内容框/居中），不改变垂直逻辑

兼容性与回退：

- 找不到参考线：完全走旧逻辑
- 参考线宽度非法（<=1）或超出矩形范围：忽略参考线，走旧逻辑

## 5. 风险与边界

- 若模板中存在大量 guide 且彼此范围重叠，方案 A 可能选到“外层 guide”，建议优先采用带 `psId` 的方案 B
- 当前 guideLayers 识别对“竖线高度”有阈值要求；如制作侧不愿画长线，可在实现阶段放宽阈值，但需防止误把装饰细线当参考线
- 本方案只改变图片对齐落地的中间文件，不触及 Photoshop 文本回写逻辑，因此不会影响文字无损替换导出

## 6. 验收方式（建议）

- 同一套商品图，在“有参考线 vs 无参考线”两种模板下对比：
  - 无参考线：结果应与当前线上一致（回归为 0）
  - 有参考线：左右边缘应稳定贴合参考线定义范围
- 通过对齐调试日志确认：
  - `guideSource`、`guideLeftX`、`guideRightX`、`targetBoxFromGuides` 存在且符合预期
