# 导出参考线冲突：变量定位高亮与配置防呆（设计）

## 背景
导出 PSD 时若同一张图片被复用于多个图片变量，但这些变量的参考线绑定（guidePick）不一致，会触发后端保护性失败。当前错误提示缺少具体变量清单，用户与管理端配置者都难以定位问题点。

## 目标
- 导出失败时给出可读的“具体图片变量清单”，并在画布上红色闪动高亮定位到变量位置（用户端与管理后台均支持）。
- 在管理后台保存配置时提前防呆：同一 Excel 图片字段被多个图片变量复用时，参考线绑定必须一致（或都不绑定），否则阻止保存并提示冲突明细。
- 调整 PSD 自动填充页默认缩放更小，避免进入后画布过大影响体验。

## 非目标
- 不改变后端导出算法与图片对齐逻辑，只增强错误可观测性与前置校验。
- 不扩展到其它工作台页或其它导出入口（仅本次涉及的用户端 PSD 自动填充与管理后台模版配置）。

## 现状
- 后端已在冲突时抛出 `err.code = DUP_IMAGE_GUIDE_MISMATCH` 且带 `err.dupImageGuideMismatch`，但 API 返回未透传该结构，前端只能看到通用 message。
- 用户端与管理后台均缺少“冲突变量的画布高亮定位”能力。
- 管理后台保存配置未校验“同一 Excel 图片字段复用但 guidePick 不一致”的情况。
- PSD 自动填充页未传 `maxInitialScale`，使用默认 0.78，进入时缩放偏大。

## 方案
### 1) 后端透传冲突结构
- `/api/template/export` 失败响应中追加 `code` 与 `dupImageGuideMismatch`（保留原 message）。
- 在后端冲突 items 中补充 `id` 与 `slotId`（来自更新指令），便于前端定位到画布变量与商品位。

### 2) 用户端导出失败高亮定位
- 当前端收到 `code=DUP_IMAGE_GUIDE_MISMATCH` 时：
  - 弹窗展示冲突变量清单（变量名/psId/商品位名/参考线 key）。
  - 自动选中第一项，并将所有冲突变量在画布上红色闪动高亮 3 秒。

### 3) 管理后台保存配置防呆与高亮定位
- 保存前校验：按 `excelFieldKey` 聚合所有图片变量，比较其 guidePick（归一化后）。
- 若同一字段存在多个不同 guidePick（含 “none” 与具体参考线混用），阻止保存：
  - 弹窗展示：字段名 + 涉及的商品位/变量名/psId/参考线 key。
  - 同时在画布上对涉及变量红色闪动高亮 3 秒，并滚动定位到第一项。

### 4) PSD 自动填充页默认缩放下调
- 给用户端 PSD 自动填充页的画布传入 `maxInitialScale`（例如 0.5），保证首次进入视野更舒适。

## 验收标准
- 导出冲突时弹窗能列出具体变量，并在画布上红色闪动高亮定位。
- 管理后台保存配置会阻止不一致的参考线复用，并提示清晰明细，同时高亮定位到对应变量。
- PSD 自动填充页进入默认缩放明显变小，减少手动调整。
