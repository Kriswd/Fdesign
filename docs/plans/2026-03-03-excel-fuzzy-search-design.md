# Excel 全表模糊搜索 + 自动定位选中（实时）设计

**目标**

- 上传 Excel 后，在数据表格上方支持“全表字段值”模糊搜索（包含匹配）。
- 搜索有命中时，自动定位到第一条命中行并选中（同步更新当前选中行）。
- 表格区域在高度限制内内部滚动，适配 1000+ 行数据。

**现状**

- 现有搜索仅按 `primaryKey` 精确匹配（`searchProduct`），且表格只渲染前 100 行，因此无法：
  - 全表模糊匹配；
  - 定位到任意命中行；
  - 对大量行在固定区域内滚动展示。

**交互设计**

- 搜索输入：实时（用户输入时即触发筛选），但加入短暂防抖以避免每个键击都全量扫描。
- 匹配规则：
  - 将查询串按空格拆分为 tokens；
  - 对每行“全字段拼接文本”的小写版本做 `includes`；
  - tokens 需全部命中（AND），以提升准确性。
- 命中行为：
  - 命中列表非空时：自动选中第一条命中行，并滚动表格定位该行；
  - 命中为空时：不改变当前选中行（避免输入过程中反复清空选择造成干扰）。
- 展示：
  - 表格仅渲染命中行列表（无查询时显示全部行）；
  - 显示统计：`命中 X / 总计 Y`。

**性能设计**

- 预计算索引：当 `rows` 或 `activeHeaders` 变化时，预计算每行的 `haystackLower`（把所有字段值拼接并小写），避免每次搜索重复拼接。
- 搜索防抖：输入变化后延迟 ~120–200ms 执行筛选，减少高频扫描。
- 渲染控制：
  - 表格容器固定高度（例如 `max-h-[60vh]`）并内部滚动；
  - 允许在未来需要时加入“最大渲染行数 + 提示”（本次先按全量渲染 1k–5k 行范围）。

**组件/数据流**

- `DataConsole` 负责：
  - 构建行索引（`useMemo`）；
  - 维护 `searchInput` 与 `debouncedQuery`；
  - 计算 `filteredRows`；
  - 选中与滚动定位（`useEffect` + `rowRef.scrollIntoView()`）；
  - 调用 `onRowSelected(row)`（由页面决定后续绑定/联动）。
- `useDataStore` 仍作为单一数据源存储 `rows/activeHeaders/currentRow`。

**验收标准**

- 上传 Excel 后，在搜索框输入任意字段内容片段，可筛出命中行。
- 有命中时自动选中第一条命中行，并滚动到可见区域。
- 1000+ 行时页面不被撑爆，表格在卡片内滚动。
