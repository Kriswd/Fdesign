# PSD 自动填充页：UI 精简与导出入口调整（设计）

## 背景
当前“PSD 自动填充”页存在历史遗留的切片/导出入口与多套编号信息，导致用户认知负担高、操作路径分散；同时画布左上角“图层数”统计恒为 0，对用户无实际价值。

## 目标
- 隐藏历史遗留的“切片与导出”整块区域，使页面聚焦在“画布 + 数据绑定 + 导出 PSD”主流程。
- 将“导出修改后的 PSD（PS 还原导出）”按钮迁移到“数据绑定”区域，降低寻找成本。
- 优化“数据绑定”中商品位的标题与绑定信息展示：展示用户命名，去掉困惑性的多套行号/序号。
- 移除无意义的“图层数统计（一直为 0）”展示。

## 非目标
- 不改动其它工作台页面（仅 PSD 自动填充页）。
- 不删除底层导出/切片逻辑，仅隐藏 UI；后续如需清理再单独做。
- 不改变导出逻辑、导出格式与后端接口。

## 现状说明
- 底部存在“数据控制台”与“切片与导出”两块：包含“切片导出 / 导出产品图 / 导出修改后的 PSD”等按钮。
- 画布左上角“X 图层”来自 `psdData.layers?.length`，但当前页并未真实填充 layers，因此恒为 0。
- 数据绑定区域中，商品位列表仅显示“商品位1”，未展示用户命名（slot.name）。绑定状态标签使用“第 N 行”显示 Excel 行号，同时下拉 option 还带 `1 -` 这种序号前缀，造成“第1行/序号1/实际 Excel 序号3”等多套编号并存的困惑。

## 方案（推荐）
### 1) 隐藏“切片与导出”整块
- 不渲染“第三行：切片与导出”整个卡片区域。
- 相关 handler（切片线、批量导出、服务端切片导出等）保留不动，以便未来需要时快速恢复。

### 2) 移动“导出修改后的 PSD”按钮到“数据绑定”区域
- 在“数据绑定”卡片标题行右侧新增主按钮：文案为“导出修改后的PSD”（沿用现有导出回写逻辑）。
- 按钮禁用规则沿用现有：导出中禁用；未选择模板时不展示或禁用（与现状一致）。
- 导出进度/错误提示逻辑不变。

### 3) 移除画布头部“图层数统计”
- 删除/隐藏“X 图层”的徽标展示。
- 若未来需要图层数：应从 PSD 解析结果中填充并明确“变量/图层”定义，但不在本次范围。

### 4) 数据绑定商品位展示优化
- 商品位标题：`商品位{index} · {slot.name}`（slot.name 为空则仅显示 `商品位{index}`）。
- 去除商品位状态标签中的“第 N 行”行号展示；用更直观的绑定提示替代：
  - 若绑定成功：优先展示 `主键: 值`；没有主键则展示第一列值；都没有则展示“已绑定（该行无可展示字段）”。
  - 未绑定：展示“未绑定（点选后在下方表格选择一条记录）”。
- 下拉选择行的 option 文案去掉 `n -` 前缀：
  - 优先展示 `主键值`；没有主键则展示 `第一列值`；都没有则展示 `第 {index+1} 行` 作为兜底（仅在下拉中出现，不在列表主视图出现）。
- pendingRow（“已选中记录”）区：不再回退展示“第 N 行”，只展示主键或第一列值，兜底展示“已选中记录”。

## 交互与可用性
- 视觉上减少多余信息，保持原有结构（画布/右侧信息/数据绑定/数据控制台）不变。
- 导出 PSD 入口靠近数据绑定，提高可发现性。

## 验收标准
- PSD 自动填充页底部不再出现“切片与导出”整块 UI。
- “导出修改后的PSD”按钮在“数据绑定”区域可见、可用，功能与之前一致。
- 画布头部不再显示“0 图层”徽标。
- 商品位列表显示用户命名（若存在），并且不再出现让用户困惑的“第1行/序号1”等重复编号展示。

## 回归验证
- 选择模板、加载画布正常。
- 商品位绑定：点选商品位 → 在下方数据控制台选一条记录完成绑定，显示文案符合预期。
- 导出：点击“导出修改后的PSD”可正常导出（含导出中禁用与错误提示）。
