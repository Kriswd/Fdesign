# Fdesign 国内 30 天内容排期

更新日期：2026-06-22

目标：让国内用户先理解 Fdesign 的真实工作流，再转化为 GitHub Star、clone、issue 和可公开案例。每天不追求多发，追求每次发布都有一个明确假设和复盘指标。

## Week 1：冷启动和问题收集

| 日期 | 渠道 | 内容 | 目标 | 复盘指标 |
| --- | --- | --- | --- | --- |
| Day 1 | 微信/小范围社群 | 社群短帖：Fdesign 已开源，求首次运行反馈 | 找 5 个真实试跑者 | 回复数、问题数、clone |
| Day 2 | V2EX | 开源项目展示帖 | 找开发者反馈 README 和安装门槛 | views、回复、Star、issue |
| Day 3 | GitHub Issues | 整理首批反馈为 FAQ issue | 把反馈沉淀到公开仓库 | issue 数、评论数 |
| Day 4 | 掘金 | 技术首文：React + Node.js 调 Photoshop | 建立开发者可信度 | 阅读、收藏、GitHub referrer |
| Day 5 | README/Docs | 补 FAQ 和 Windows 环境诊断 | 降低试跑门槛 | clone / issue 质量 |
| Day 6 | B 站 | 60-90 秒工作流演示 | 让设计/运营看懂产品特点 | 播放、收藏、评论问题 |
| Day 7 | 复盘 | 运行指标脚本，写周报 | 决定下周内容重心 | stars、views、clones、referrers |
