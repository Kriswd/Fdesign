# Fdesign Launch Copy Benchmark

Date: 2026-06-16

This benchmark summarizes public GitHub peers inspected during the 1000-star growth design. It uses paraphrase only and focuses on reusable launch patterns.

## bjango/Bjango-Actions

- First-screen proof asset: strong hero image before the project title.
- Value proposition style: collection of practical Photoshop/design utilities.
- Quick-start clarity: direct download link and version history are visible early.
- Community/support path: documentation links are grouped immediately under the intro.
- Fdesign takeaway: lead with a real visual proof asset and make the first action obvious.

## lohriialo/photoshop-scripting-python

- First-screen proof asset: simple visual plus a teaching-oriented title.
- Value proposition style: explains why scripting matters before diving into code.
- Quick-start clarity: installation and hello-world example appear near the top.
- Community/support path: examples and API reference are discoverable.
- Fdesign takeaway: keep a short English summary and explain the automation concept for search traffic.

## Investigamer/Proxyshop

- First-screen proof asset: centered showcase image and badges.
- Value proposition style: specific niche output, not generic Photoshop automation.
- Quick-start clarity: requirements are explicit, including Windows and Photoshop constraints.
- Community/support path: support/community link is prominent.
- Fdesign takeaway: be concrete about ecommerce PSD production and state Windows + Photoshop early.

## joonaspaakko/Batch-Mockup-Smart-Object-Replacement-photoshop-script

Readable project name: Batch Mockup Smart Object Replacement.

- First-screen proof asset: animated workflow preview.
- Value proposition style: one clear batch-processing job.
- Quick-start clarity: documentation links are very short and direct.
- Community/support path: Discussions are enabled.
- Fdesign takeaway: a future GIF/video can help explain the workflow, but the current screenshot is enough for V3.0 launch.

## alisaitteke/photoshop-mcp

- First-screen proof asset: badges and standalone UI screenshot.
- Value proposition style: developer-facing automation bridge with modern AI keywords.
- Quick-start clarity: one-command startup example is visible early.
- Community/support path: package/repo links make adoption easy.
- Fdesign takeaway: keep developer credibility below the ecommerce value proposition instead of making it the headline.

## Fdesign Copy Rules

- Use the concrete transformation: Excel product data -> batch PSD deliverables.
- Use the real workbench screenshot as proof.
- Do not claim unverified time savings.
- Keep the shop link secondary to open-source quick start.
- Invite stars after the value is clear.
- Make Windows + Photoshop requirements impossible to miss.
